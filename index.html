Git and GitHub

Following is the some of the git commands and why it is used for.

    Firstly open the terminal in the vs code and do the Following

  • git init: Initializes a new Git repository in the current directory. This creates a new .git directory where Git stores its metadata and configuration.
  • git add: Adds changes in the working directory to the staging area, preparing them to be committed. You can add specific files (git add ) or add all changes (git add .).
  • git status: Displays the current state of the working directory and staging area. It shows which files have been modified, which are staged for commit, and which are not tracked by Git.
  • git commit: Records changes in the staging area to the repository, creating a new commit. You must provide a commit message describing the changes (git commit -m "message").
  • git log: Displays a chronological list of commits in the repository. This includes commit hashes, authors, dates, and commit messages.
  • git reset --hard: Resets the staging area and working directory to match the most recent commit, discarding any changes or staged files.
  • git reset --soft: Resets the HEAD to a previous commit, but leaves the changes staged. This allows you to recommit the changes with a new commit message.
  • git branch: Lists all branches in the repository. By default, it shows the local branches, but you can add -a to see all branches (local and remote).
  • git push: Uploads local repository content to a remote repository. It's used to share local changes with others or update a remote branch.
  • git clone: Creates a local copy of a remote repository. It copies the entire repository, including all branches and commit history.
  • git pull: Fetches changes from a remote repository and merges them into the current branch. It's used to update your local repository with changes made by others.
  • git merge: Combines the changes from one branch into another. It's typically used to integrate feature branches into the main branch.
